    Graphyn Architecture Audit — 2026-06-18

    Conducted via kotlin-multiplatform-expert (godmode orchestrator) + kotlin-multiplatform-audit.

    Module health

    Module Status
    :core clean (WorkflowGraphValidator was 193 lines — split this session)
    :core:designsystem new — all files under limit
    :editor-api clean — all files < 62 lines
    :plugin-api clean — all files < 71 lines
    :app:shared 6 files over 150-line ceiling — split this session
    :plugins:* clean
    :server clean

    Findings

    F1 — Design system fragmentation (Medium) — Fixed

    5 canvas/shell components used MaterialTheme directly despite GraphynDsTheme being the designated system. Three parallel color sources (GraphynDs, AppTheme, MaterialTheme) created potential drift on theme change.

    Files fixed: - GraphynZoomControls.kt — FilledTonalIconButton → Foundation Box+clickable + GraphynDs - GraphynCanvasTelemetryOverlay.kt — Surface/Button/MaterialTheme → Foundation + GraphynDs - GraphynNodePickerPopup.kt — Surface/DropdownMenuItem/HorizontalDivider → Foundation + GraphynDs - GraphynCanvasHint.kt — MaterialTheme.colorScheme → GraphynDs - GraphynCanvasBackdrop.kt — MaterialTheme.colorScheme.surfaceVariant/surface → GraphynDs

    F2 — File size violations (Medium) — Fixed

    6 files exceeded the 150-line CLAUDE.md ceiling.

    File Before After split
    GraphynEditorState.kt 245 ~105 + 3 action files
    GraphynMinimapDebugger.kt 176 ~145 + MiniMapColors file
    GraphynNodePortDots.kt 167 ~35 + 2 port-dot files
    GraphynMinimapMath.kt 165 ~80 + GraphynMinimapLayout file
    GraphynCanvasSurface.kt 164 ~90 + GraphynNodeLayer file
    GraphynTheme.kt 152 ~65 + 2 split files

    F3 — WorkflowGraphValidator over limit in :core (Low) — Fixed

    193 lines. Extracted validateConnections and validateRequiredInputs into WorkflowConnectionValidation.kt.

    F4 — core:designsystem not yet consumed by editor (Low) — Deferred

    AppTheme is scaffolded and bridged inside GraphynDsTheme so both AppTheme.Local and LocalGraphynDsColors are provided simultaneously. Editor components still use GraphynDs.* accessors. Coexistence is stable; full migration is a separate phase once the design system is battle-tested.

    Passing

    • Module boundaries: :ui never imports :data; no DTOs or entities cross module boundaries
    • MVI pattern: all mutations flow through dispatch(GraphynEditorIntent) on GraphynEditorState
    • expect/actual: only 2 pairs (PlatformInfo, GraphynSettingsStore), both well-scoped
    • Test coverage: 22 test files across commonTest + jvmTest; interactive elements all tagged

    Fix order applied this session

    1. Migrate 5 components off MaterialTheme → GraphynDs.* (F1)
    2. Split GraphynEditorState → state + 3 action extension files (F2)
    3. Split GraphynTheme → presets + appearance state + theme composable (F2)
    4. Split GraphynMinimapMath → layout+bounds + coordinate math (F2)
    5. Split GraphynMinimapDebugger → debugger + colors (F2)
    6. Split GraphynNodePortDots → coordinator + per-port composables (F2)
    7. Split GraphynCanvasSurface → surface + node layer (F2)
    8. Split WorkflowGraphValidator → validator + connection/input validators (F3)
