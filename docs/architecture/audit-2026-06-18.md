## Findings

### F1 — Design system fragmentation (Medium) — Fixed
5 canvas/shell components used `MaterialTheme` directly despite `GraphynDsTheme` being the designated system.
Three parallel color sources (GraphynDs, AppTheme, MaterialTheme) created potential drift on theme change.

**Files fixed:**
- `GraphynZoomControls.kt` — FilledTonalIconButton → Foundation Box+clickable + GraphynDs
- `GraphynCanvasTelemetryOverlay.kt` — Surface/Button/MaterialTheme → Foundation + GraphynDs
- `GraphynNodePickerPopup.kt` — Surface/DropdownMenuItem/HorizontalDivider → Foundation + GraphynDs
- `GraphynCanvasHint.kt` — MaterialTheme.colorScheme → GraphynDs
- `GraphynCanvasBackdrop.kt` — MaterialTheme.colorScheme.surfaceVariant/surface → GraphynDs

### F4 — core:designsystem not yet consumed by editor (Low) — Deferred
`AppTheme` is scaffolded and bridged inside `GraphynDsTheme` so both `AppTheme.Local` and
`LocalGraphynDsColors` are provided simultaneously. Editor components still use `GraphynDs.*`
accessors. Coexistence is stable; full migration is a separate phase once the design system
is battle-tested.
