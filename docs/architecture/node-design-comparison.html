Verdict — recommended design for Graphyn

Take ComfyUI's port system (edge dots + type colour + inline widgets) and combine it with Blender's category header (coloured bar = node family) and keep Graphyn's execution badge + output preview — nothing else does that. Add a compact flag on NodeSpec to opt into the n8n circle for simple service nodes.

① Port dots on card edges (High) ② Dot colour by WorkflowType (High) ③ Content-driven card height (Medium) ④ compact flag → icon-circle variant (Medium) ⑤ Inline widget for unconnected scalars (Medium) ⑥ Category colour on header bar (Low)

Plugin customization — what works today vs. what's missing

Partial

✓ NodeSpec registration works

A plugin calls registerNodeSpec(spec) to declare port names, port types, and default values. The palette and inspector both react automatically.

✓ Executor registration works

A plugin calls registerExecutor(type, fn). The engine resolves it at runtime. No coupling between the canvas and execution logic.

✓ Custom inspector panel works

A plugin registers an EditorPanelFactory for a node type. When that node is selected, the inspector renders the plugin's @Composable instead of the default panel.

✗ Custom canvas card missing

GraphynNodeLayer hardcodes GraphynNodeCard for every node. Plugins cannot supply a custom @Composable for how their node looks on the canvas.

✗ Node icon missing

NodeSpec has no icon field. The palette shows label text only. Plugins cannot attach a painter/vector to their node type.

✗ Node category / colour missing

NodeSpec has no category or color field. Every node renders with the same accent bar colour. Blender/ComfyUI both use this to group families visually.

What to add — NodeSpec fields

icon: Painter? = null
category: String = "general"
color: Color? = null
compact: Boolean = false

What to add — canvas factory

Add NodeCanvasFactory to editor-api (mirrors EditorPanelFactory). Register in GraphynEditorPlugin. GraphynNodeLayer resolves it and falls back to GraphynNodeCard when absent.

Plugin that customizes its canvas card — target API
object MyPlugin : GraphynPlugin {
    override fun register(registrar: GraphynPluginRegistrar) {
        registrar.registerNodeSpec(NodeSpec(
            type     = "my.node",
            label    = "My Node",
            icon     = MyIcons.spark,       // NEW
            category = "AI",               // NEW
            color    = Color(0xFF6D5AFF),  // NEW
            compact  = false,              // NEW
            inputs   = listOf(...),
            outputs  = listOf(...),
        ))
        registrar.registerCanvasCard("my.node") { ctx ->   // NEW
            MyCustomNodeCard(ctx.node, ctx.spec, ctx.state)
        }
        registrar.registerPanel("my.node") { ctx ->
            MyInspectorPanel(ctx)
        }
    }
}
What each editor does best
ComfyUI Port colour system + inline widget that disappears on connect
Blender Editable fields on the canvas, category header colour, collapse mode
ReactFlow Maximum flexibility — any React component is a node card
Node-RED Lowest friction for non-developers; colour = category identity
n8n Two-tier system (detail card + icon circle) scales to 100+ node types
LiteGraph Proven at scale (ComfyUI), typed ports, widget-on-canvas pattern
Recommended Graphyn design — steal the best
From ComfyUI — edge dots, port type colour, widget-on-unconnected-port
From Blender — category colour bar, content-driven height, collapse toggle
From ReactFlow — NodeCanvasFactory so plugins own their card UI
From n8n — compact (icon-circle) flag for simple service nodes
Keep from Graphyn — execution badge, live output preview (nobody else has this)
Add to NodeSpec: icon · category · color · compact
Add to editor-api: NodeCanvasFactory
